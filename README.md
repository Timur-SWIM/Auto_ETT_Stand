# Стенд управления температурой на микроконтроллере Milandr MDR32F9Q2I

Проект представляет собой аппаратно-программный стенд для измерения и регулирования температуры с использованием микроконтроллера **Milandr MDR32F9Q2I** (ядро Cortex‑M3). Стенд включает в себя:
- Измерение температуры с помощью АЦП и датчика LM35
- PID‑регулятор для поддержания заданной температуры
- ШИМ (PWM) для управления нагревательным элементом
- ЦАП (DAC) с DMA для генерации тестовых сигналов
- USB CDC (Virtual COM Port) для связи с компьютером
- Отладочный вывод и приём команд через USB

Проект собран в среде Keil µVision 5 и использует стандартную библиотеку периферии (SPL) от Milandr.

## Основные функции

### 1. Измерение температуры
- АЦП работает в режиме с DMA, канал подключён к датчику LM35 (10 мВ/°C)
- Опорное напряжение 3,3 В, разрешение 12 бит (0–4095)
- Функция `ADC_ToTemp()` преобразует код АЦП в температуру (°C)
- Усреднение нескольких отсчётов для повышения точности

### 2. PID‑регулятор
- Реализован классический ПИД‑регулятор с коэффициентами Kp, Ki, Kd
- Заданная температура устанавливается через USB‑команду
- Выход регулятора – скважность ШИМ (0–100 %)
- Ограничение по температуре (20–120 °C) и по скважности (10–100 %)

### 3. ШИМ (PWM)
- Генерируется таймером на выводе PB0
- Частота ШИМ ≈ 62,5 кГц (при SysClk = 80 МГц)
- Скважность изменяется в реальном времени по сигналу PID

### 4. USB CDC (Virtual COM Port)
- Эмуляция последовательного порта через USB
- Приём команд с компьютера (текстовые команды)
- Отправка текущей температуры и отладочных сообщений
- Поддержка функций `printf` через `USB_PrintDebug()`

### 5. ЦАП (DAC) с DMA
- ЦАП работает в режиме с DMA для генерации произвольных сигналов
- Таймер запускает преобразования с заданной частотой
- Возможность вывода тестовых сигналов (синус, меандр и т.д.)

## Сборка и запуск

### Требования
- Keil µVision 5 (или более поздняя)
- Пакет поддержки Milandr MDR32FxQI (версия 1.3.2)
- Отладочный программатор (J‑Link, U‑Link) или внутрисхемный программатор
- Целевое устройство: отладочная плата с микроконтроллером MDR32F9Q2I

### Шаги сборки
1. Откройте файл `stand.uvprojx` в Keil µVision.
2. Убедитесь, что в проекте выбрано устройство **MDR32F9Q2I**.
3. Соберите проект (F7).
4. Загрузите прошивку в микроконтроллер (Flash → Download).
5. Подключите USB‑кабель к порту микроконтроллера (или отладочной платы).

### Настройка аппаратуры
- Датчик LM35 подключён к выводу PA0 (канал АЦП0)
- ШИМ‑выход (управление нагревателем) – PB0
- ЦАП‑выход – PA4 (при использовании)
- USB‑разъём – стандартный USB Type‑B или Micro‑USB (в зависимости от платы)

## Работа с USB

После подключения к компьютеру система определит виртуальный COM‑порт. Для взаимодействия можно использовать любой терминал (PuTTY, Tera Term, screen и т.д.) с настройками:
- Скорость: не имеет значения (USB CDC)
- 8 бит данных, 1 стоп‑бит, без контроля чётности

### Доступные команды
- `TEMP` – запрос текущей температуры
- `SET xxx` – установка целевой температуры (xxx – целое число, °C)
- `PID ON/OFF` – включение/выключение PID‑регулятора
- `PWM xx` – прямая установка скважности (xx – 0–100 %)
- `HELP` – вывод списка команд

Пример сеанса:
```
> TEMP
Текущая температура: 25 °C
> SET 80
Установлена температура: 80 °C
> PID ON
PID включен
```

## Настройка параметров

Основные параметры системы задаются в заголовочных файлах:

### `inc/pwm.h`
```c
#define T_MIN 20          // минимальная температура, °C
#define T_MAX 120         // максимальная температура, °C
#define PWM_PERIOD 1281   // период ШИМ (ARR+1)
#define PWM_MIN_DUTY 10   // минимальная скважность, %
#define PWM_MAX_DUTY 100  // максимальная скважность, %

#define PID_KP 1.0f       // пропорциональный коэффициент
#define PID_KI 0.1f       // интегральный коэффициент
#define PID_KD 0.01f      // дифференциальный коэффициент
```

### `inc/adc.h`
```c
#define VREF 3300         // опорное напряжение, мВ
#define LM35_SENSITIVITY 10  // чувствительность LM35, мВ/°C
```

## Отладка

В проекте активен макрос `DEBUG` (файл `usb.h`), что позволяет выводить отладочные сообщения через USB. Для отключения отладочного вывода закомментируйте строку `#define DEBUG`.

Также можно использовать встроенный отладчик Keil с окнами Watch, Memory и Logic Analyzer (для анализа ШИМ и сигналов ЦАП).

## Возможные расширения

1. **Веб‑интерфейс** – добавление Ethernet/Wi‑Fi модуля для удалённого контроля.
